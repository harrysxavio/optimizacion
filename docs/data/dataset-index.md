# Dataset Index вҖ” Slotting Optimization Engine

**Last updated:** 2026-05-28  
**Status:** Active (Phase 6 вҖ” synthetic + processed diagnostics + scoring + scenarios + optimization + simulation)

## Source datasets

All source datasets are generated by `scripts/generate_sample_data.py` and
stored as CSV in `data/synthetic/`. Each dataset is deterministic (seed=42
by default).

### 1. SKU Master (`skus`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `sku_id` | string | Yes | Unique SKU identifier (PK) |
| `category` | string | Yes | Product category |
| `subcategory` | string | No | Product subcategory |
| `unit_volume` | float | Yes | Volume per unit (cmВі) |
| `unit_weight` | float | Yes | Weight per unit (kg) |
| `rotation_class` | string | Yes | A/B/C/D rotation class |
| `avg_daily_demand` | float | Yes | Average daily demand (units) |

**Constraints:** `sku_id` unique, `unit_volume` > 0, `unit_weight` > 0, rotation_class in {A,B,C,D}.

### 2. Zones (`zones`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `zone_id` | string | Yes | Unique zone identifier (PK) |
| `zone_type` | string | Yes | Type (picking, reserve, bulk, cross_dock) |
| `priority_level` | integer | Yes | 1=closest to dispatch |
| `distance_to_dispatch` | float | Yes | Relative distance to dispatch |
| `max_volume_capacity` | float | Yes | Maximum volume capacity |
| `max_weight_capacity` | float | Yes | Maximum weight capacity |

**Constraints:** `zone_id` unique, capacities > 0, priority >= 1.

### 3. Locations (`locations`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `location_id` | string | Yes | Unique location identifier (PK) |
| `zone_id` | string | Yes | FK вҶ’ zones.zone_id |
| `aisle` | string | Yes | Aisle identifier |
| `rack` | string | No | Rack identifier |
| `level` | string | No | Level within rack |
| `max_volume_capacity` | float | Yes | Volume capacity for this location |
| `max_weight_capacity` | float | Yes | Weight capacity for this location |

**Constraints:** `location_id` unique, capacities > 0, zone_id must exist in zones.

### 4. Inventory (`inventory`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `sku_id` | string | Yes | FK вҶ’ skus.sku_id |
| `location_id` | string | Yes | FK вҶ’ locations.location_id |
| `units_on_hand` | integer | Yes | Current units stored |
| `occupied_volume` | float | Yes | Volume currently used |
| `occupied_weight` | float | Yes | Weight currently used |

**Constraints:** All fields >= 0. SKU and location must exist in their masters.

### 5. Orders (`orders`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `order_id` | string | Yes | Unique order identifier (PK) |
| `order_date` | date | Yes | Order date |
| `channel` | string | No | Sales channel |

**Constraints:** `order_id` unique.

### 6. Order Lines (`order_lines`)

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `order_id` | string | Yes | FK вҶ’ orders.order_id |
| `sku_id` | string | Yes | FK вҶ’ skus.sku_id |
| `quantity` | integer | Yes | Quantity ordered |

**Constraints:** `quantity` > 0. FK references must exist.

## Derived datasets (features, diagnostics, scoring, scenarios, and optimization)

Generated by `scripts/build_features.py` and stored in `data/processed/`.

### 7. Slotting Features (`slotting_features.parquet`)

One row per SKU with all analytical features merged. Columns include:

| Field | Description |
|-------|-------------|
| `sku_id` | SKU identifier |
| `category`, `subcategory` | Product taxonomy |
| `unit_volume`, `unit_weight` | Physical dimensions |
| `rotation_class` | Rotation class (A/B/C/D) |
| `avg_daily_demand` | Generated daily demand |
| `total_demand` | Aggregated demand from order lines |
| `order_count` | Number of orders containing this SKU |
| `total_volume` | Total occupied volume in inventory |
| `total_weight` | Total occupied weight in inventory |
| `alignment_flag` | вҡ пёҸ Non-prescriptive alignment heuristic |
| `alignment_score` | вҡ пёҸ Non-prescriptive alignment score [0,1] |

### 8. Location Utilization (`location_utilization.csv`)

| Field | Description |
|-------|-------------|
| `location_id` | Location identifier |
| `volume_utilization_pct` | Used volume as % of capacity (capped at 100) |
| `weight_utilization_pct` | Used weight as % of capacity (capped at 100) |
| `sku_count_in_location` | Number of distinct SKUs in this location |
| `over_capacity` | Boolean flag for over-capacity locations |

### 9. Zone Utilization (`zone_utilization.csv`)

| Field | Description |
|-------|-------------|
| `zone_id` | Zone identifier |
| `zone_type` | Zone classification |
| `priority_level` | Priority level |
| `distance_to_dispatch` | Distance to dispatch |
| `avg_volume_util_pct` | Mean volume utilisation across locations |
| `avg_weight_util_pct` | Mean weight utilisation across locations |
| `location_count` | Number of locations in zone |
| `over_capacity_location_count` | Number of over-capacity locations |

### 10. Slotting Diagnostics (`slotting_diagnostics.csv`)

SKU-level descriptive diagnostic flags for high-demand poor placement and low-demand/slow-rotation SKUs in premium zones. Thresholds are `inferred / pending confirmation`.

### 11. Location Diagnostics (`location_diagnostics.csv`)

Location-level descriptive diagnostics for overutilization, underutilization, SKU density, category mix, and average utilization.

### 12. Zone Diagnostics (`zone_diagnostics.csv`)

Zone-level descriptive diagnostics for utilization status and low-demand/slow-rotation SKU presence in premium zones.

### 13. Category Diagnostics (`category_diagnostics.csv`)

Category-level spread and grouping indicators when category and placement fields are available.

### 14. Diagnostic Summary (`diagnostic_summary.csv`)

Metric/value summary of Phase 2 diagnostic counts with threshold notes.

### 15. Slotting Opportunity Scores (`slotting_opportunity_scores.csv`)

Action-level Phase 3 prioritization scores generated by `scripts/run_scoring.py` from Phase 2 diagnostics. Contains `entity_type`, `entity_id`, `candidate_action`, `opportunity_score`, `priority`, `rank`, explanation fields, and `inferred / pending confirmation` caveats.

### 16. Priority Recommendation Queue (`priority_recommendation_queue.csv`)

Sorted human-review queue generated from opportunity scores. It contains candidate review labels only and must not be interpreted as an optimal move plan.

### 17. Scoring Summary (`scoring_summary.csv`)

Metric/value summary of Phase 3 output volume, priority counts, score statistics, and inferred scoring configuration snapshot.

### 18. Scenario Comparison (`scenario_comparison.csv`)

Top-N analytical what-if comparison rows generated by `scripts/run_scenarios.py` from Phase 3 scores. Contains scenario rank, weighted scenario score, scenario description, entity/action context, Phase 3 score context, assumption state, and explicit non-optimization caveat.

### 19. Scenario Action Mix (`scenario_action_mix.csv`)

Candidate-action mix by scenario with selected counts, shares, high-priority counts, average opportunity score, and total weighted opportunity. This is an analytical summary only, not an execution plan.

### 20. Scenario Summary (`scenario_summary.csv`)

Scenario-level metrics for selected count, top-N, total weighted opportunity, average scores, high-priority count, SKU/zone emphasis, candidate-action coverage, dominant action, assumption state, and config snapshot.

### 21. Optimization Assignments (`optimization_assignments.csv`)

Controlled Phase 5 SKU-to-zone assignment rows generated by `scripts/run_optimization.py`. Contains selected SKU, target zone, logical zone slot, opportunity/scenario context, cost drivers, solver method, inferred assumption state, and explicit no-execution/no-location-feasibility caveat.

### 22. Optimization Summary (`optimization_summary.csv`)

Metric/value summary for selected SKUs, assigned rows, candidate zone slots, target zones used, assignment costs, solver method, inferred assumption state, caveat, and config snapshot.

### 23. Optimization Cost Matrix (`optimization_cost_matrix.csv`)

Transparent SKU-zone-slot matrix used by the Phase 5 assignment prototype. Each row records cost drivers for auditability; it is not a physical slot feasibility table.

### 24. Simulation Summary (`simulation_summary.csv`)

High-level Phase 6 simulation metrics. Contains scenario run date, input description, number of SKUs/orders/lines/zones, current and optimized distance and time, distance and time saved, average improvement %, Gini coefficients (before/after), throughput estimates (orders/shift before/after, gain %), and the simulation caveat.

### 25. Simulation Travel Aggregate (`simulation_travel_aggregate.csv`)

Aggregated travel metrics: total distance (current/optimized), total time (current/optimized), distance saved, time saved, average improvement %. One row per run.

### 26. Simulation Zone Impact (`simulation_zone_impact.csv`)

Zone-level workload detail: current picks per zone, optimized picks per zone, pick change (absolute and %), and Gini coefficient describing workload balance across zones.

### 27. Simulation Throughput Scenarios (`simulation_throughput_scenarios.csv`)

Throughput estimates under 3 scenarios (optimistic, balanced, conservative). Each scenario includes total time saved (s), orders/shift before, orders/shift after, throughput gain %, and the multiplier used.

### 28. Simulation Order Detail (`simulation_order_detail.csv`)

Per-order simulation breakdown: order ID, number of lines, current distance and time, optimized distance and time, distance saved, time saved, current zone IDs visited, candidate zone IDs visited. Useful for debugging and manual inspection of edge cases.
